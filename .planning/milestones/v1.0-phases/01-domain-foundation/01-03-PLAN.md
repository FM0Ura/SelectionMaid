    Method `process(self, input: RawInput) -> ExtractionResult` implements the pipeline with four bounded try/except blocks (one per port call), following exactly the pattern from D-16:

    Step 1 — extract:
      try: raw = self._extractor.extract(input)
      except SelectionMaidError: raise
      except Exception as e: raise ExtractionError(message=f"Extraction failed: {e}", cause=e) from e

    Step 2 — filter:
      try: filtered = self._filter.filter(raw)
      except SelectionMaidError: raise
      except Exception as e: raise FilterError(message=f"Filtering failed: {e}", cause=e) from e

    Step 3 — chunk:
      try: chunks_list = self._chunker.chunk(filtered.content)
      except SelectionMaidError: raise
      except Exception as e: raise ChunkingError(message=f"Chunking failed: {e}", cause=e) from e

    Step 4 — enrich:
      try: metadata = self._enricher.enrich(raw, chunks_list)
      except SelectionMaidError: raise
      except Exception as e: raise EnrichmentError(message=f"Enrichment failed: {e}", cause=e) from e

    Return `ExtractionResult(metadata=metadata, chunks=tuple(chunks_list))`. Note: the ChunkerPort returns `list[DocumentChunk]` but ExtractionResult.chunks is `tuple[DocumentChunk, ...]` — convert with `tuple(chunks_list)`.

    Do NOT import `docling`, `fastapi`, `pydantic`, or any third-party package. Do NOT use `asyncio` or `async def`.

|-----------|----------|-----------|-------------|-----------------|
