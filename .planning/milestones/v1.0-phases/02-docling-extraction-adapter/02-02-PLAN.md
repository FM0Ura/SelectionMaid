<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RawInput.path → converter.convert() | Path from domain object passed to Docling; path traversal or malicious file content enters at this boundary |
| ConversionResult → RawDocument | Docling output mapped to domain type; content from document conversion crosses into domain here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | RawInput.path passed to converter.convert() | accept | Path sanitization is Phase 6 responsibility (tempfile creation); adapter trusts the path it receives per RESEARCH.md Security Domain |
| T-02-05 | DoS | converter.convert() blocking call | mitigate | ThreadPoolExecutor with future.result(timeout=self._timeout_seconds) prevents indefinite blocking; timeout translated to ExtractionTimeoutError |
| T-02-06 | Tampering | Malicious PDF with zip bomb or deeply nested structures | mitigate | ExtractionError raised for any Docling exception (covers bomb-triggered OOM/runtime errors); process does not crash — error is returned to caller |
| T-02-07 | DoS | Password-protected PDF | accept | Docling raises RuntimeError → caught by except Exception → ExtractionError returned; no crash |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck completed for docling in RESEARCH.md Package Legitimacy Audit — no new installs in this plan |
</threat_model>
