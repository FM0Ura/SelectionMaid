# Phase 8 — UI Design Contract

> Visual and interaction contract for Phase 8: Backend CORS + Project Scaffold.
> Generated by gsd-ui-researcher. This phase establishes the foundational design
> tokens consumed by all downstream frontend phases (9-13).

---

## Context Note

Phase 8 delivers zero user-facing application screens. Its single visual deliverable
is a proof-of-life: one shadcn-vue `Button` component rendered in `App.vue` to confirm
that Tailwind CSS v4 + shadcn-vue + dark mode are wired correctly.

All design tokens declared here are the baseline contract for Phases 9-13. Downstream
phases MUST NOT introduce new base tokens without updating this spec first.

**Source of decisions:** 08-CONTEXT.md (D-09 through D-11), STATE.md §Accumulated
Context → Decisions, ROADMAP.md Phase 8 success criteria.

## Color

Dark mode is fixed for v2.0 (`class="dark"` hardcoded on `<html>`). All values are
dark-mode surfaces. Light mode is out of scope.

shadcn-vue dark mode tokens (CSS variables set by the CLI init):

| Role | CSS Variable | Approximate Hex | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | `#09090b` (zinc-950) | Page background, main surface |
| Secondary (30%) | `--card` / `--muted` | `#18181b` (zinc-900) | Cards, panels, sidebar, nav |
| Accent (10%) | `--primary` | `#fafafa` (zinc-50) | Specific reserved elements (see below) |
| Destructive | `--destructive` | `#7f1d1d` (red-900 dark) | Destructive actions only |

**Accent (`--primary`) reserved for:**
- Primary CTA button fill (the proof-of-life `Button` in Phase 8)
- The "Upload Document" CTA button label (Phase 10)
- Active state of the drop zone border (Phase 10)
- Copy-to-clipboard button hover state (Phase 12)

Accent is NOT used for: section headings, body text, card borders, metadata labels,
or decorative elements.

**Note:** shadcn-vue CLI generates CSS variable tokens in `src/assets/index.css`.
Executor must not override these variables with hardcoded hex values — use the CSS
variable names throughout.

## Copywriting Contract

Phase 8 has one visible text element: the proof-of-life Button label in `App.vue`.

| Element | Copy | Source |
|---------|------|--------|
| Proof-of-life button label | "Upload Document" | discretion default — matches Phase 10 primary CTA for continuity |
| Page title (`<title>`) | "SelectionMaid" | project name |
| `App.vue` scaffold heading | "SelectionMaid is ready." | discretion default — confirms scaffold works |

**Empty state:** Not applicable in Phase 8 (no data-driven views). Defined in Phase 10.

**Error state:** Not applicable in Phase 8 (no API calls). Defined in Phase 9.

**Destructive actions:** None in Phase 8.

**Rationale for button copy:** The proof-of-life Button uses "Upload Document" rather
than a placeholder like "Click Me" so that Phase 10 can reuse the same component
without a copy change. This makes the scaffold directly forward-compatible.

## Proof-of-Life Visual Specification

`<div class="flex flex-col items-center justify-center min-h-screen gap-6">`

toolchain. All real UI structure begins in Phase 10.
