## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline code padding |
| sm | 8px | Button icon gap, metadata cell inner spacing, copy/download button gap |
| md | 16px | Card body padding, chunk header padding, form element spacing |
| lg | 24px | Section gaps within ResultView, DropZone internal padding |
| xl | 32px | Gap between ResultView sections (MetadataCard → ChunkList) |
| 2xl | 48px | Page-level vertical breathing room (main padding-y) |

Exceptions:
- Touch-safe icon-only buttons (download per-chunk, copy button): minimum 32px × 32px hit area (size="sm" in shadcn = h-8 = 32px) — acceptable for desktop-only internal tool.
- ChunkCard content area: max-h-[400px] — fixed cap, not on the 4px grid.
- MetadataCard stat cells: `px-4 py-2` (16px horizontal, 8px vertical — both in the standard set).

**shadcn token nomenclature note:** In shadcn, `--accent` is a **surface background token** (used for hover states on menu items, table headers, code block backgrounds) — it is NOT the brand accent color. When implementing, use `--primary` / `text-primary` / `border-primary` for the purple brand color; use `--accent` only as a surface fill for code blocks and table headers.

## Download Feature — Functional Contract

### File Name Pattern

```
{slugified-source_filename}-chunks.md
```

Slugification rules:
- Lowercase all characters
- Replace spaces and special characters with `-`
- Collapse multiple `-` into one
- Strip extension from original filename before slugifying
- Re-append `-chunks.md`

Example: `"Calendário de Provas 2026.pdf"` → `"calendario-de-provas-2026-chunks.md"`

Function to add to `frontend/src/lib/formatters.ts`:
```typescript
export function slugifyFilename(filename: string): string {
  const base = filename.replace(/\.[^.]+$/, '') // strip extension
  return base
    .toLowerCase()
    .normalize('NFD')
    .replace(/[̀-ͯ]/g, '') // strip diacritics
    .replace(/[^a-z0-9]+/g, '-')
    .replace(/^-+|-+$/g, '')
}
```

Source: D-18 from 14-CONTEXT.md.

### Generated .md Format

```markdown
---
title: {metadata.title || metadata.source_filename}
language: {metadata.language}
doc_type: {metadata.doc_type}
ingested_at: {metadata.ingested_at}
chunk_count: {metadata.chunk_count}
---

# Chunk 1
<!-- pages: {chunk.page_start}-{chunk.page_end} | words: {chunk.word_count} | section: {chunk.section_title} -->

{chunk.content}

---

# Chunk 2
...
```

Source: D-19 from 14-CONTEXT.md.

### Per-Chunk Download Format

Single chunk file name: `chunk-{N}-{slugified-section-title}.md`

Content: just the raw `chunk.content` string — no front-matter, no separator.
